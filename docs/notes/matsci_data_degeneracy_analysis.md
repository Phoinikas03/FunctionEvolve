# MatSci 数据集采样退化诊断

本文记录对 `nnheui/llm-srbench` 中 `lsr_synth_matsci` split 的一次数据诊断。触发问题是：`MatSci5` 的多个候选表达式在 train/test/OOD 上取得极低 NMSE，但结构等价验证无法匹配 ground truth。诊断结论是：这不是单个候选式的问题，而是整个 `matsci` split 存在明显的一维采样退化；此外，若干 Arrhenius 型因子在当前温度范围内变化很小，导致真实结构项难以被数值误差区分。

## 核心结论

`matsci` split 一共 25 个方程：

```text
MatSci0, MatSci1, MatSci2, MatSci3, MatSci4,
MatSci5, MatSci6, MatSci7, MatSci8, MatSci9,
MatSci10, MatSci11, MatSci12, MatSci13, MatSci14,
MatSci16, MatSci18, MatSci19, MatSci20, MatSci21,
MatSci22, MatSci23, MatSci24, MatSci25, MatSci28
```

所有方程的输入变量都是：

```text
epsilon, T
```

但实际样本并没有覆盖二维输入空间，而是几乎完全落在同一条直线上：

```text
T ≈ 273 + 500 * epsilon
```

这意味着名义上的二维符号回归问题，在数据上退化成了一维函数近似问题。任何 `T` 相关结构都可能被改写成纯 `epsilon` 函数，并在采样点上取得很低 NMSE。

## 变量范围

以 `MatSci5` 为例，两个输入变量范围如下：

| split | epsilon range | T range |
|---|---:|---:|
| train | `0` ～ `0.5399879813` | `273` ～ `542.9940186` |
| test | `0.0018003600` ～ `0.5365073085` | `273.9001770` ～ `541.2536621` |
| OOD | `0.5401080251` ～ `0.6000000238` | `543.0540161` ～ `573` |

这也解释了 OOD 的含义：它不是独立改变 `T` 或 `epsilon`，而是继续沿同一条近似直线向右外推。

## MatSci5 的结构退化示例

`MatSci5` 的 ground truth 为：

```text
61.34686906400979*epsilon*(-0.05076522329965729*(T - 278.14787512499896) + 1)
+ 98.57939495694947*epsilon**0.3178497443603504*exp(-49.45582326747874/(8.314*T))
+ epsilon**1.450294566141342*2.304053848821008*(T - 278.14787512499896)
```

展开后包含如下结构项：

```text
T*epsilon
epsilon
epsilon^0.31785 * exp(-5.94899/T)
T*epsilon^1.45029
epsilon^1.45029
```

由于实际样本满足：

```text
T ≈ 273 + 500*epsilon
```

关键耦合项会退化为：

```text
T*epsilon^q
≈ (273 + 500*epsilon)*epsilon^q
= 273*epsilon^q + 500*epsilon^(q+1)
```

对 `q=1.450294566141342`，在真实样本上用 `[epsilon^q, epsilon^(q+1)]` 拟合 `T*epsilon^q` 的相对 L2 残差只有：

```text
train: 2.875e-08
test : 2.905e-08
OOD  : 3.550e-08
```

这意味着 `T*epsilon^1.45` 在数据上几乎不可与两个纯 `epsilon` 幂项区分。

进一步把 `T=273+500*epsilon` 代入 `MatSci5` GT，可得到近似的一元形式：

```text
77.3788*epsilon
-1557.1438*epsilon^2
+98.5794*epsilon^0.31785*exp(-5.94899/(273 + 500*epsilon))
-11.8610*epsilon^1.45029
+1152.0269*epsilon^2.45029
```

这解释了为什么日志中一些不含完整 GT 结构的候选式可以取得极低 NMSE：它们不是恢复了真实二维结构，而是在这条一维采样轨迹上做了高精度函数近似。

## Arrhenius 因子变化范围

`matsci` 中含 Arrhenius 型 `exp(-c/T)` 的方程有 8 个：

```text
MatSci1, MatSci3, MatSci5, MatSci10,
MatSci11, MatSci12, MatSci23, MatSci25
```

这些因子在 train/test 上的相对变化约为 `0.11%` 到 `2.06%`；在 OOD 上更小，约为 `0.006%` 到 `0.108%`。

所以对 Arrhenius 项而言，“变化很弱”是指：在当前 `T` 范围内，`exp(-c/T)` 的数值几乎可以被常数吸收。例如 `MatSci5` 中：

```text
exp(-5.9485/T)
```

在 train 上只从 `0.978446` 变化到 `0.989105`，相对变化约 `1.09%`。因此：

```text
epsilon^p * exp(-c/T)
```

在样本点上主要表现为：

```text
常数 * epsilon^p
```

## 其它非线性项

并不是所有非线性项都“变化很小”。例如：

```text
log(epsilon + 1): 0 ～ 0.432
exp(-epsilon):   0.583 ～ 1
sin(epsilon):    0 ～ 0.514
sin(T - c):      约 -1 ～ 1
```

这些项本身有明显变化。但由于 `T` 与 `epsilon` 被绑定为一维关系，`sin(T-c)` 也只是 `sin(273 + 500*epsilon - c)` 的一元函数；它不能提供真正独立的 `T` 维度信息。

此外，Gaussian 型项：

```text
exp(-(T - T0)^2)
```

不是变化小，而是高度稀疏或饱和。对应方程包括 `MatSci12`、`MatSci13`、`MatSci16`。在 train 样本中：

| equation | atom | fraction > 1e-6 | fraction > 1e-3 | fraction > 0.1 |
|---|---|---:|---:|---:|
| MatSci12 | `exp(-(T - 367.02302414249573)**2)` | 0.0265 | 0.0190 | 0.0112 |
| MatSci13 | `exp(-(T - 298.0016449216105)**2)` | 0.0270 | 0.0187 | 0.0105 |
| MatSci16 | `exp(-(T - 325.3097844170149)**2)` | 0.0278 | 0.0190 | 0.0100 |

也就是说，这类因子在绝大多数样本点上接近 0，只有约 1% 到 3% 的点有明显贡献。这同样会削弱结构辨识能力。

## 对符号回归结果的影响

### 1. NMSE 低不代表结构正确

NMSE 衡量的是采样点上的数值误差，不衡量结构等价性。当前 `matsci` 数据几乎是一维采样，因此许多错误结构可以在采样线上补偿真实结构。

例如 `MatSci5` 中，GT 需要：

```text
T*epsilon^1.45029
```

但在当前数据上它几乎等价于：

```text
273*epsilon^1.45029 + 500*epsilon^2.45029
```

所以候选式可以缺少真实耦合项，却仍然通过纯 `epsilon` 幂项取得极低 NMSE。

### 2. `T` 相关结构不可独立识别

由于所有样本几乎满足 `T=273+500*epsilon`，下面这些结构在数据上都可能相互混淆：

```text
T * f(epsilon)
(273 + 500*epsilon) * f(epsilon)
273*f(epsilon) + 500*epsilon*f(epsilon)
g(epsilon)
```

因此如果候选式只用纯 `epsilon` 函数拟合得很好，不能说明数据不需要 `T`；只能说明当前采样不能区分 `T` 的独立作用。

### 3. OOD 并没有解决二维外推

当前 OOD 仍沿同一条直线：

```text
train/test: epsilon ≈ 0 ～ 0.54, T ≈ 273 ～ 543
OOD       : epsilon ≈ 0.54 ～ 0.60, T ≈ 543 ～ 573
```

这只是沿一维轨迹继续外推，不是对 `T` 和 `epsilon` 的独立干预。因此 OOD NMSE 低也不能证明恢复了真实二维结构。

## 建议

如果目标是发现真实公式结构，当前 `matsci` split 不适合作为可靠结构恢复基准。建议至少做以下一种改动：

1. 重新采样，使 `epsilon` 和 `T` 独立变化，例如二维网格或 Latin hypercube。
2. 构造干预测试：固定 `epsilon` 改变 `T`，以及固定 `T` 改变 `epsilon`。
3. 对含 Arrhenius 项的任务扩大或重设 `T` 范围，使 `exp(-c/T)` 有可观变化。
4. 对 Gaussian 型 `exp(-(T-T0)^2)` 项增加 `T0` 附近样本，否则该项在大部分点上近似为 0。
5. 结构验证和报告中不要只依赖 NMSE；应明确标注 `matsci` 的输入共线性风险。

## 总结

全 `matsci` split 的主要问题是：

```text
T ≈ 273 + 500*epsilon
```

这使二维结构恢复问题退化为一维函数近似问题。Arrhenius 型 `exp(-c/T)` 因子在当前范围内普遍只变化约 `0.1%` 到 `2%`，进一步削弱了结构辨识能力。对于 `MatSci5` 这类任务，候选式 NMSE 极低但不匹配 GT 并不矛盾；这恰恰是数据不可辨识性的表现。
